# 📣 Red Zone App (SA-Projectxz / speakup-api)

**Red Zone App** คือแพลตฟอร์มเว็บแอปพลิเคชันสำหรับแจ้งเตือนและระบุพิกัดพื้นที่เสี่ยงภัยคุกคามทางเพศและจุดอับสายตา เพื่อให้ผู้ใช้งานสามารถรายงานเหตุการณ์ (แบบไม่ระบุตัวตน - Anonymous)[cite: 2, 7] และช่วยสร้างแผนที่ความปลอดภัยร่วมกันในชุมชน 

นอกจากนี้ตัวแอปพลิเคชันยังมีระบบจำลอง AI Agent (P Diddy AI) คอยให้คำปรึกษาเบื้องต้นเกี่ยวกับการปฏิบัติตัวและข้อกฎหมายอีกด้วย

## ⚙️ หลักการทำงานของแต่ละส่วน (How It Works)

### 1. หน้าบ้านและระบบแผนที่ (`index.html`)[cite: 2]
ส่วนแสดงผลและโต้ตอบกับผู้ใช้งาน (Client-Side) มีฟีเจอร์และการทำงานหลักดังนี้:
* **การวาดแผนที่ (Map Rendering):** แสดงแผนที่แบบ Interactive ผ่าน **Leaflet.js** ดึงข้อมูลแผนที่ฐานจาก OpenStreetMap โดยมีพิกัดเริ่มต้นศูนย์กลางอยู่ที่กรุงเทพฯ[cite: 2]
* **การคำนวณและจับกลุ่มจุดเสี่ยง (Coordinate Grouping Algorithm):** 
  * ระบบจะดึงข้อมูลพิกัดที่มีการแจ้งเหตุทั้งหมดจาก API หลังบ้าน[cite: 2]
  * ทำการตรวจสอบและจัดกลุ่ม (Grouping) หมุดที่อยู่ใกล้เคียงกันด้วยค่าความคลาดเคลื่อน (Threshold) ไม่เกิน `0.0005` (หรือรัศมีประมาณ 50 เมตรรอบพิกัดนั้น ๆ)[cite: 2]
  * หากมีจุดเสี่ยงหลายจุดอยู่ในรัศมีเดียวกัน ระบบจะนำมารวมกลุ่มเพื่อแสดงผลเป็น **"วงกลมแสดงความเสี่ยงขนาดใหญ่เพียงวงเดียว"** เพื่อหลีกเลี่ยงปัญหาหมุดทับซ้อนกันจนมองไม่เห็นรายละเอียด (หากหมุดใดหมุดหนึ่งในกลุ่มเป็นระดับสีแดง วงกลมของกลุ่มนั้นจะกลายเป็นสีแดงตามไปด้วย)[cite: 2]
* **แถบแสดงรายละเอียดเหตุการณ์ (Incident Feed Panel):** เมื่อผู้ใช้คลิกที่วงกลมสีแดงหรือสีเหลืองบนแผนที่ แถบด้านขวาจะแสดงรายการเหตุการณ์ทั้งหมดที่เกิดขึ้นในบริเวณนั้น โดยเรียงลำดับเวลาจาก "ใหม่ล่าสุด" ไป "เก่าที่สุด"[cite: 2]
* **ระบบจำลอง AI Chat Agent (P Diddy AI):** ระบบแชทอัตโนมัติฝั่ง Client-side ที่ทำงานด้วยเทคนิคจับคู่คำสำคัญ (Keyword Matching) เช่น หากมีคำว่า "เดินตาม", "ตาม", "กฎหมาย", "แจ้งความ", หรือ "ตำรวจ" ระบบจะวิเคราะห์และเลือกข้อความตอบกลับเพื่อแนะนำสิทธิ์และข้อกฎหมายเบื้องต้นแก่ผู้ใช้[cite: 2]

### 2. เซิร์ฟเวอร์หลังบ้าน (`index.js`)
ทำหน้าที่เป็น Web API เชื่อมต่อกับระบบจัดการฐานข้อมูลคลาวด์ (Cloud Database) มีกลไกสำคัญดังนี้:
* **ความปลอดภัย (Security & CORS):** ควบคุมความปลอดภัยในการส่งข้อมูลผ่านระบบ CORS เพื่อเปิดโอกาสให้ฝั่ง Frontend สามารถ Fetch ข้อมูลไปใช้งานได้ และเชื่อมต่อไปยัง Cloud Database (TiDB Cloud AWS) แบบเข้ารหัสลับผ่าน SSL
* **การปกป้องตัวตนผู้รายงาน (Anonymous Flow):** เมื่อเซิร์ฟเวอร์ได้รับข้อมูลการแจ้งเหตุผ่านเมธอด `POST` ระบบจะทำการสุ่มชื่อนามแฝงขึ้นมาใหม่เสมอ เช่น `user_` ตามด้วยเลขสุ่ม 6 หลัก (ตัวอย่างเช่น `user_492019`)[cite: 7] บันทึกลงไปแทนชื่อจริงของผู้แจ้ง เพื่อรักษาความปลอดภัยและความเป็นส่วนตัวสูงสุด[cite: 2, 7]
* **API Endpoints:**
  * `POST /api/incidents` : ใช้ในการรับข้อมูลรายงานจุดเสี่ยงใหม่จากหน้าบ้านเพื่อบันทึกลงฐานข้อมูล[cite: 7]
  * `GET /api/incidents` : ดึงข้อมูลพิกัดและรายงานทั้งหมดส่งคืนให้หน้าบ้านนำไปเรนเดอร์ลงแผนที่[cite: 7]

### 3. แฟ้มบันทึก Dependencies (`package.json`)[cite: 4]
บรรจุไลบรารีสำคัญที่เป็นกำลังหลักของระบบหลังบ้าน ได้แก่[cite: 4]:
* `express` : สำหรับรัน HTTP Server และจัดการ REST API Route[cite: 4, 5]
* `mysql2` : ไดรเวอร์สำหรับรับส่งคำสั่ง SQL ไปยังฐานข้อมูล TiDB Cloud[cite: 4, 5, 7]
* `cors` : จัดการการเข้าถึงข้ามโดเมนระหว่างพอร์ตหน้าบ้านและพอร์ตหลังบ้าน[cite: 4, 5, 7]
