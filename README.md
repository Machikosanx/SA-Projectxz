# 📣 Red Zone App (SA-Projectxz / speakup-api)

**Red Zone App** คือแพลตฟอร์มเว็บแอปพลิเคชันสำหรับแจ้งเตือนและระบุพิกัดพื้นที่เสี่ยงภัยคุกคามทางเพศและจุดอับสายตา เพื่อให้ผู้ใช้งานสามารถรายงานเหตุการณ์ (แบบไม่ระบุตัวตน - Anonymous) และช่วยสร้างแผนที่ความปลอดภัยร่วมกันในชุมชน 

นอกจากนี้ตัวแอปพลิเคชันยังมีระบบจำลอง AI Agent (P Diddy AI) คอยให้คำปรึกษาเบื้องต้นเกี่ยวกับการปฏิบัติตัวและข้อกฎหมายอีกด้วย

## ⚙️ หลักการทำงานของแต่ละส่วน (How It Works)

### 1. หน้าบ้านและระบบแผนที่ (`index.html`)
ส่วนแสดงผลและโต้ตอบกับผู้ใช้งาน (Client-Side) มีฟีเจอร์และการทำงานหลักดังนี้:
* **การวาดแผนที่ (Map Rendering):** แสดงแผนที่แบบ Interactive ผ่าน **Leaflet.js** ดึงข้อมูลแผนที่ฐานจาก OpenStreetMap โดยมีพิกัดเริ่มต้นศูนย์กลางอยู่ที่กรุงเทพฯ
* **การคำนวณและจับกลุ่มจุดเสี่ยง (Coordinate Grouping Algorithm):** 
  * ระบบจะดึงข้อมูลพิกัดที่มีการแจ้งเหตุทั้งหมดจาก API หลังบ้าน
  * ทำการตรวจสอบและจัดกลุ่ม (Grouping) หมุดที่อยู่ใกล้เคียงกันด้วยค่าความคลาดเคลื่อน (Threshold) ไม่เกิน `0.0005` (หรือรัศมีประมาณ 50 เมตรรอบพิกัดนั้น ๆ)
  * หากมีจุดเสี่ยงหลายจุดอยู่ในรัศมีเดียวกัน ระบบจะนำมารวมกลุ่มเพื่อแสดงผลเป็น **"วงกลมแสดงความเสี่ยงขนาดใหญ่เพียงวงเดียว"** เพื่อหลีกเลี่ยงปัญหาหมุดทับซ้อนกันจนมองไม่เห็นรายละเอียด (หากหมุดใดหมุดหนึ่งในกลุ่มเป็นระดับสีแดง วงกลมของกลุ่มนั้นจะกลายเป็นสีแดงตามไปด้วย)
* **แถบแสดงรายละเอียดเหตุการณ์ (Incident Feed Panel):** เมื่อผู้ใช้คลิกที่วงกลมสีแดงหรือสีเหลืองบนแผนที่ แถบด้านขวาจะแสดงรายการเหตุการณ์ทั้งหมดที่เกิดขึ้นในบริเวณนั้น โดยเรียงลำดับเวลาจาก "ใหม่ล่าสุด" ไป "เก่าที่สุด"
* **ระบบจำลอง AI Chat Agent (P Diddy AI):** ระบบแชทอัตโนมัติฝั่ง Client-side ที่ทำงานด้วยเทคนิคจับคู่คำสำคัญ (Keyword Matching) เช่น หากมีคำว่า "เดินตาม", "ตาม", "กฎหมาย", "แจ้งความ", หรือ "ตำรวจ" ระบบจะวิเคราะห์และเลือกข้อความตอบกลับเพื่อแนะนำสิทธิ์และข้อกฎหมายเบื้องต้นแก่ผู้ใช้

### 2. เซิร์ฟเวอร์หลังบ้าน (`index.js`)
ทำหน้าที่เป็น Web API เชื่อมต่อกับระบบจัดการฐานข้อมูลคลาวด์ (Cloud Database) มีกลไกสำคัญดังนี้:
* **ความปลอดภัย (Security & CORS):** ควบคุมความปลอดภัยในการส่งข้อมูลผ่านระบบ CORS เพื่อเปิดโอกาสให้ฝั่ง Frontend สามารถ Fetch ข้อมูลไปใช้งานได้ และเชื่อมต่อไปยัง Cloud Database (TiDB Cloud AWS) แบบเข้ารหัสลับผ่าน SSL
* **การปกป้องตัวตนผู้รายงาน (Anonymous Flow):** เมื่อเซิร์ฟเวอร์ได้รับข้อมูลการแจ้งเหตุผ่านเมธอด `POST` ระบบจะทำการสุ่มชื่อนามแฝงขึ้นมาใหม่เสมอ เช่น `user_` ตามด้วยเลขสุ่ม 6 หลัก (ตัวอย่างเช่น `user_492019`) บันทึกลงไปแทนชื่อจริงของผู้แจ้ง เพื่อรักษาความปลอดภัยและความเป็นส่วนตัวสูงสุด
* **API Endpoints:**
  * `POST /api/incidents` : ใช้ในการรับข้อมูลรายงานจุดเสี่ยงใหม่จากหน้าบ้านเพื่อบันทึกลงฐานข้อมูล
  * `GET /api/incidents` : ดึงข้อมูลพิกัดและรายงานทั้งหมดส่งคืนให้หน้าบ้านนำไปเรนเดอร์ลงแผนที่

### 3. แฟ้มบันทึก Dependencies (`package.json`)
บรรจุไลบรารีสำคัญที่เป็นกำลังหลักของระบบหลังบ้าน ได้แก่:
* `express` : สำหรับรัน HTTP Server และจัดการ REST API Route
* `mysql2` : ไดรเวอร์สำหรับรับส่งคำสั่ง SQL ไปยังฐานข้อมูล TiDB Cloud
* `cors` : จัดการการเข้าถึงข้ามโดเมนระหว่างพอร์ตหน้าบ้านและพอร์ตหลังบ้าน

### 💻 ขั้นตอนฝั่งเซิร์ฟเวอร์หลังบ้าน (Backend Server)

1. เปิดโปรแกรม Terminal หรือ Command Prompt แล้วเข้าไปยังโฟลเดอร์ของ API หลังบ้าน:
   ```bash
   cd speakup-api
2. ทำการติดตั้ง Dependencies ทั้งหมดที่จำเป็น:
   ```bash
   npm install
3. การตั้งค่าฐานข้อมูล (Database Connection):
   * ปัจจุบันระบบในไฟล์ `index.js` ถูกตั้งค่าให้เชื่อมต่อกับคลาวด์ดาต้าเบสออนไลน์ของ TiDB Cloud เรียบร้อยแล้ว
   * หากคุณต้องการเปลี่ยนไปทดสอบกับฐานข้อมูล MySQL จำลองในเครื่องส่วนตัว (Localhost เช่น XAMPP, phpMyAdmin) ให้เข้าไปแก้ไขที่ตัวแปรสร้าง Connection ใน `index.js` เป็นสเปกโฮสต์ของคุณ
4. รันคำสั่งเริ่มทำงานเซิร์ฟเวอร์หลังบ้าน:
   ```bash
   node index.js
## 📱 ขั้นตอนฝั่งหน้าเว็บ (Frontend UI)
1. การเปิดใช้งานหน้าเว็บ:
   * ดับเบิ้ลคลิกเปิดไฟล์ `index.html` บนเว็บบราวเซอร์ เช่น Google Chrome, Safari หรือ Microsoft Edge ได้โดยตรง
   * หรือเปิดผ่าน Extension "Live Server" บน VS Code เพื่อการพัฒนาที่สะดวกยิ่งขึ้น
2. การตั้งค่าเชื่อมต่อกับเซิร์ฟเวอร์ API:
   * หากกำลังทดสอบระบบหลังบ้านในเครื่องของตัวเอง (Localhost) ให้เปิดไฟล์ index.html แล้วแก้ไข URL สำหรับ Fetch ข้อมูล:
     * ค้นหาคำว่า https://sa-projectxz.onrender.com/api/incidents
     * เปลี่ยนตัวนี้ให้ชี้ไปยังเซิร์ฟเวอร์โลคอลของคุณแทน: http://localhost:3000/api/incidents
   * หากต้องการใช้งานผ่านระบบคลาวด์เวอร์ชัน Live ที่จัดเตรียมไว้แล้ว สามารถเปิดใช้งานและทดสอบได้ทันทีโดยไม่ต้องแก้ไข URL ใด ๆ
## Back End Server :
1. TiDB Cloud (tidbcloud.com) คือ แพลตฟอร์มผู้ให้บริการระบบจัดการฐานข้อมูลบนคลาวด์ (Cloud Database) ที่ช่วยให้นักพัฒนาสามารถรันฐานข้อมูลออนไลน์ระดับ Enterprise ได้ง่ายและรวดเร็ว:
   * https://tidbcloud.com/
2. Vercel คือ แพลตฟอร์มคลาวด์ (Cloud Platform) ที่ออกแบบมาสำหรับ Deploy (อัปโหลด) หน้าเว็บฝั่งหน้าบ้าน (Frontend) ให้ขึ้นไปออนไลน์บนโลกอินเทอร์เน็ตได้ง่าย รวดเร็ว และมีประสิทธิภาพสูงมาก:
   * https://vercel.com/
3. Render (render.com) คือ แพลตฟอร์มคลาวด์แบบ PaaS (Platform-as-a-Service) ที่ช่วยให้นักพัฒนาสามารถอัปโหลดและรันระบบหลังบ้าน (Backend Server), ฐานข้อมูล (Database), หรืองานประมวลผลเบื้องหลัง (Background Worker) ขึ้นไปออนไลน์ได้ง่ายๆ โดยแทบไม่ต้องตั้งค่า Server (เช่น Linux, Nginx, Docker) ด้วยตัวเองเลย:
   * https://Render.com/
